# ⚙️ Дополнительные функции

## Pairlist Selector

ML-инструмент для оптимизации списка торговых пар.

### Страница Pairlist Selector (`/pairlist-selector`)

Позволяет запустить оптимизацию пар для выбранной стратегии в Docker-контейнере.

**Параметры запуска:**

| Параметр | Описание |
|----------|----------|
| Strategy | Стратегия для оптимизации |
| Mode | `ml_training`, `fullbacktest_batch`, `fullbacktest_individual` |
| N Pairs | Количество пар для отбора |
| Download Days | Дней данных для загрузки |
| Backtest Days | Дней для бэктеста |
| Max Pairs | Максимум пар для анализа |

**Результаты (страница Pairlist Results):**
- Список завершённых job'ов с метаданными
- Отобранные пары с метриками
- Статус выполнения (running, completed, failed)

## Alerts & Уведомления

Система алертов отслеживает события и уведомляет пользователя.

### Типы алертов

| Тип | Уровень | Описание |
|-----|---------|----------|
| Bot Health | CRITICAL | Бот недоступен |
| Bot Health | WARNING | Задержки API, ошибки |
| System | INFO | Бот обнаружен, обновление |
| Rate Limit | WARNING | Достигнут лимит запросов биржи |

### Управление алертами

Страница Alerts (`/alerts`):

- **Mark Read** — отметить выбранные как прочитанные
- **Mark All Read** — отметить всё как прочитанное
- **Dismiss** — скрыть алерты
- **Delete Dismissed** — удалить скрытые алерты
- **Clear Rate Limit** — снять блокировку rate limit для бота

Фильтры: по типу, уровню, боту, статусу прочтения.

## Системные настройки

Страница Settings (`/settings`) — раздел **System Settings**:

| Настройка | Описание | По умолчанию |
|-----------|----------|-------------|
| Refresh Interval | Интервал обновления данных (сек) | 30 |
| Theme | Тёмная/светлая тема | dark |
| Notifications | Включить уведомления | true |
| Dashboard Layout | Сетка/список | grid |
| Alert Critical | Показывать критические алерты | true |
| Alert Warning | Показывать предупреждения | true |
| Alert Info | Показывать информационные | false |
| Discovery Host IP | IP для обнаружения (при ручной настройке) | — |
| API Username/Password | Креды по умолчанию для Freqtrade API | — |

## Пользователи (User Management)

Страница доступна только для администраторов (`/users`).

### Роли

| Роль | Права |
|------|-------|
| **Admin** | Полный доступ: управление ботами, пользователями, настройками |
| **Operator** | Управление ботами, просмотр портфеля |
| **ReadOnly** | Только просмотр дашборда и метрик |

### Управление

- **Create User** — создать нового пользователя
- **Edit** — изменить имя, роль, пароль
- **Delete** — удалить пользователя
- У каждого пользователя свои сохранённые настройки (тема, уведомления и т.д.)

## Биллинг (WATA)

Интеграция с платёжным провайдером **WATA** для управления подписками.

**Функции:**
- Просмотр текущего плана и подписки
- История платежей (аудит)
- Создание checkout-сессии для оплаты
- Portal return для управления подпиской
- Webhook-уведомления о платежах

**Планы тарифов:**
- Бесплатный starter с триал-периодом
- Платные планы с расширенными лимитами

Настройка биллинга производится через переменные окружения:
- `WATA_API_TOKEN` — токен API WATA
- `BILLING_ENFORCED=true` — включить принудительную проверку подписки

## Multi-Tenancy

FreqDash поддерживает мультитенантность для SaaS-развёртывания:

- Каждый tenant получает отдельную схему в PostgreSQL
- Данные изолированы между tenant'ами
- Tenant определяется через заголовок `X-Tenant-Slug` или URL-префикс `/t/{slug}/`
- Регистрация нового tenant'а через страницу Signup

### Журнал аудита

Все операции биллинга и администрирования логируются в таблицу `billing_audit_logs` для последующего анализа.
