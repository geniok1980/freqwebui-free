# ⚙️ Дополнительные функции

## Селектор пар (Pairlist Selector)

Инструмент машинного обучения для оптимизации списка торговых пар.

### Страница «Селектор пар» (`/pairlist-selector`)

Позволяет запустить оптимизацию пар для выбранной стратегии в Docker-контейнере.

**Параметры запуска:**

| Параметр | Описание |
|----------|----------|
| Стратегия | Стратегия для оптимизации |
| Режим | `ml_training`, `fullbacktest_batch`, `fullbacktest_individual` |
| Количество пар | Сколько пар отобрать |
| Дней для загрузки | Сколько дней данных загрузить |
| Дней для теста | За какой период тестировать |
| Максимум пар | Верхняя граница для анализа |

**Результаты (страница результатов селектора пар):**
- Список завершённых задач с метаданными
- Отобранные пары с метриками
- Статус выполнения (выполняется, завершено, ошибка)

## Уведомления

Система отслеживает события и оповещает пользователя.

### Типы уведомлений

| Тип | Уровень | Описание |
|-----|---------|----------|
| Состояние бота | КРИТИЧЕСКИЙ | Бот недоступен |
| Состояние бота | ПРЕДУПРЕЖДЕНИЕ | Задержки API, ошибки |
| Система | ИНФО | Бот обнаружен, обновление |
| Лимит запросов | ПРЕДУПРЕЖДЕНИЕ | Достигнут лимит запросов биржи |

### Управление уведомлениями

Страница «Уведомления» (`/alerts`):

- **Отметить прочитанным** — отметить выбранные как прочитанные
- **Отметить все прочитанными** — отметить всё
- **Скрыть** — скрыть уведомления
- **Удалить скрытые** — очистить скрытые записи
- **Снять лимит запросов** — снять блокировку для бота

Фильтры: по типу, уровню, боту, статусу прочтения.

## Пользователи

Страница доступна только для администраторов (`/users`).

### Роли

| Роль | Права |
|------|-------|
| **Администратор** | Полный доступ: управление ботами, пользователями, настройками |
| **Оператор** | Управление ботами, просмотр портфеля |
| **Только чтение** | Только просмотр дашборда и метрик |

### Управление

- **Создать пользователя** — добавить нового
- **Изменить** — изменить имя, роль, пароль
- **Удалить** — удалить пользователя
- У каждого пользователя свои сохранённые настройки (тема, уведомления и т.д.)

## Биллинг (WATA)

Интеграция с платёжным провайдером **WATA** для управления подписками.

**Возможности:**
- Просмотр текущего тарифа и подписки
- История платежей
- Создание сессии оплаты
- Управление подпиской через портал
- Вебхуки о платежах

**Тарифы:**
- Бесплатный стартовый с пробным периодом
- Платные тарифы с расширенными лимитами

Настройка биллинга через переменные окружения:
- `WATA_API_TOKEN` — токен API WATA
- `BILLING_ENFORCED=true` — включить принудительную проверку подписки

## Мультиарендность

FreqDash поддерживает работу нескольких клиентов (арендаторов) на одном экземпляре для SaaS-развёртывания:

- Каждый арендатор получает отдельную схему в PostgreSQL
- Данные изолированы между арендаторами
- Арендатор определяется через заголовок `X-Tenant-Slug` или префикс URL `/t/{id}/`
- Регистрация нового арендатора через страницу регистрации

### Журнал аудита

Все операции биллинга и администрирования записываются в таблицу `billing_audit_logs` для последующего анализа.
