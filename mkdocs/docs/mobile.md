# 📱 Мобильное приложение

FreqDash Mobile — нативное приложение для iOS и Android на базе Expo (React Native). Доступно после сборки через EAS Build.

## Сборка

### Требования

- Node.js 20+
- Expo CLI (`npm install -g eas-cli`)
- Аккаунт Expo (expo.dev)
- Доступ к EAS Build

### Сборка APK (Android)

```bash
cd mobile
npx eas build --platform android --profile preview --local
```

### Сборка IPA (iOS)

```bash
cd mobile
npx eas build --platform ios --profile preview --local
```

## Экраны приложения

| Экран | Вкладка | Описание |
|-------|---------|----------|
| **Дашборд** | 📊 | Сводка портфеля, метрики ботов |
| **Финансы** | 💰 | Криптовалюты, акции, новости |
| **Обнаружение** | 🔍 | Автообнаружение ботов |
| **Агенты** | 🤖 | Веса агентов, управление |
| **Уведомления** | 🔔 | Системные оповещения |

### Экран «Дашборд»

- Краткая сводка портфеля (общая прибыль, баланс, активные боты)
- Список ботов со статусом
- Нажатие на бота → переход к деталям

### Экран «Карточка бота»

- Информация о боте (биржа, стратегия, статус)
- Текущие метрики (прибыль, сделки, баланс)
- Кнопки управления (если доступно)

### Экран «Финансовые данные»

- Список криптовалют с ценами
- Изменение за 24 часа
- Обновление свайпом вниз

### Экран «Обнаружение»

- Статус автообнаружения
- Ручной запуск сканирования
- Добавление бота вручную

### Экран «Агенты»

- Веса сигналов по рыночным режимам
- Статистика агента (процент побед, сделки)

### Экран «Уведомления»

- Список активных уведомлений
- Фильтрация по типу и уровню
- Отметить прочитанным / скрыть

## Настройка подключения

При первом запуске приложение запросит адрес сервера FreqDash:

1. Экран настройки → введите адрес вашего сервера
   - Например: `http://192.168.1.100:15000`
2. Войдите с вашими учётными данными
3. Данные авторизации сохраняются в памяти устройства

## Технический стек

- **Expo SDK 57** — основа приложения
- **React 19** + **React Native 0.86**
- **React Navigation 7** — навигация (нижние вкладки + стек)
- **TanStack React Query 5** — кэширование и загрузка данных
- **Zustand 5** — управление состоянием
- **TypeScript 6**
