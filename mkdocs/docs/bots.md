# 🤖 Управление ботами

## Добавление ботов

### Автообнаружение

FreqDash автоматически сканирует доступные боты Freqtrade каждые 60 секунд двумя способами:

**Поиск через Docker**
- Сканирует запущенные Docker-контейнеры через Docker socket (`/var/run/docker.sock`)
- Ищет контейнеры с образом `freqtradeorg/freqtrade`
- Автоматически определяет URL, порт и статус контейнера
- Требуется установка метки `com.freqtrade.bot_name` на контейнере

**Поиск через файловую систему**
- Сканирует файловую систему в поиске баз данных `tradesv3.sqlite`
- Пути сканирования по умолчанию: `/opt/freqtrade/*/user_data`
- Подходит для установок Freqtrade без Docker

**Проверить статус обнаружения:**
- Страница Обнаружение (`/discovery`)
- Отображает: включён ли поиск в Docker, поиск по файлам, время последнего сканирования, время следующего

**Ручной запуск сканирования:**
Нажмите **«Запустить сканирование»** на странице Обнаружение.

### Ручная регистрация

Если бот не обнаружился автоматически:

1. Перейдите в **Боты → Добавить бота**
2. Заполните:
   - **Имя** — уникальное имя бота (латиница, цифры, `-`, `_`)
   - **URL API** — например, `http://192.168.1.100:8080`
   - **Имя пользователя / Пароль** — данные API из конфига Freqtrade (`api_server.username` / `api_server.password`)
3. Нажмите **Проверить подключение**
4. Нажмите **Сохранить**

## Страница «Боты» (`/freqtrade-bots`)

Отображает таблицу всех зарегистрированных ботов:

| Колонка | Описание |
|---------|----------|
| Имя | Название бота |
| Биржа | Биржа, на которой торгует |
| Стратегия | Активная стратегия |
| Состояние | Состояние (🟢 В порядке, 🟡 Замедлен, 🔴 Недоступен) |
| Прибыль | Текущая доходность |
| Сделок | Количество сделок |
| Среда | Docker / Вручную / Без Docker |
| Последний раз | Время последнего контакта |

**Фильтрация:**
- По среде выполнения (Docker, Вручную, Without Docker)
- По состоянию
- По бирже
- По стратегии
- По меткам

## Карточка бота (`/bots/:id`)

Вкладки:
- **Обзор** — сводка: баланс, открытые позиции, прибыль, график капитала
- **Сделки** — история сделок с фильтрацией по времени, паре, типу
- **Аналитика** — метрики: процент успешных сделок, фактор прибыли, Шарпа, просадка
- **Состояние** — время работы, задержки, ошибки подключения, лимиты запросов
- **Настройки** — конфигурация бота (`config.json`, чтение и запись)

## Деплой нового бота

FreqDash может развернуть нового бота Freqtrade в Docker одной кнопкой:

1. **Боты → Развернуть бота**
2. Выберите **стратегию** из списка доступных
3. Настройте:
   - **Имя бота** — уникальное имя
   - **Порт на хосте** — внешний порт (1024–65535)
   - **Тестовый режим** — включить режим симуляции
4. Нажмите **Развернуть**

Система создаст:
- Docker-контейнер `freqtrade-bot-{имя}`
- Конфиг с API-сервером (автоматическая генерация JWT-секрета)
- Стратегию с корректировками для спотового режима (`can_short = False`)
- Монтирование в `/opt/freqtrade_secrets/{имя_бота}/`

## Управление конфигурацией

На странице бота → вкладка **Настройки**:

- **Прочитать конфиг** — просмотр текущего `config.json` бота
- **Редактировать конфиг** — изменение конфига (JSON)
- **Сохранить конфиг** — запись изменений в контейнер или на диск

Для ботов в Docker поиск конфига выполняется в:
1. `/freqtrade/user_data/config.json`
2. `/freqtrade/user_data/config/config.json`
3. `/freqtrade/user_data/configs/config.json`

## Управление состоянием

- **Запустить / Остановить / Перезапустить** — управление через Docker API
- **Проверка состояния** — принудительная проверка
- **Удалить** — удаление бота из системы (без удаления Docker-контейнера)

## Лимиты запросов

FreqDash отслеживает лимиты запросов к биржам через анализ логов ботов:

- Автоматическое обнаружение ошибок 429
- Отображение на странице «Уведомления»
- Ручное снятие блокировки: кнопка **Снять лимит** на странице бота или в Уведомлениях
- Лимиты автоматически сбрасываются через 10 минут без новых срабатываний
